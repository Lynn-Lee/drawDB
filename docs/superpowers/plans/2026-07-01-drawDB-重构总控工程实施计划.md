# drawDB 重构总控工程实施计划

> **给 agent 执行者的要求：** 具体执行本计划时，必须使用 `superpowers:subagent-driven-development`（推荐）或 `superpowers:executing-plans` 按任务推进。本文是总控计划，阶段开工前还需要生成对应 Phase 的细粒度实施计划。

**目标：** 将 drawDB 的产品重构方案转化为可分阶段交付的工程计划，在保持“无需账号、本地优先”核心体验可用的前提下，系统性提升安全性、数据模型可靠性、导入导出正确性、编辑器体验、可访问性、性能，以及后续可选的账号/团队/云端能力。

**架构策略：** 按 6 个阶段推进。Phase 0 已建立安全和测试底座；Phase 1 稳定 diagram domain model、command history 和本地持久化；Phase 2 强化导入导出；Phase 3 优化编辑器 UX 和 accessibility；Phase 4 治理 bundle 和大图性能；Phase 5 在明确能力开关后增加可选账号、团队和云端同步，不强制登录。

**技术栈：** React 18、Vite 8、Dexie/IndexedDB、Semi UI、Monaco、Lexical、Vitest、Testing Library、Playwright、axe、npm audit、Docker/nginx。

---

## 1. 范围与执行规则

本文是整体工程总控计划，不直接替代每个阶段的施工单。每个阶段开工前，必须在 `docs/superpowers/plans/` 下生成对应中文阶段实施计划，并细化到文件、测试、验证命令和提交粒度。

硬性产品约束：

- 每个阶段结束后，本地无账号编辑必须仍然可用。
- 登录、团队空间、云端同步默认属于 Phase 5，除非产品负责人明确调整优先级。
- 分享必须是用户显式触发的网络动作，不能自动上传图表数据。
- SQL、DBML、JSON、分享链接内容都按非可信输入处理。
- 每个阶段至少通过 `npm run lint`、`npm run build` 和该阶段相关测试。

当前产品方案源文档：

- `docs/drawDB-重构优化产品研发方案.md`

## 12. 风险登记

1. Domain 迁移可能破坏已有 IndexedDB 图表。
   - 缓解：保留旧结构 fixtures，所有读取先 normalize，保留全量本地备份路径。

2. SQL 导入导出支持范围可能无限扩张。
   - 缓解：明确支持语法，不支持语法给出清晰 issue，不追求 100% SQL parser。

3. Canvas 性能优化可能改变交互语义。
   - 缓解：重构前补 drag、select、connect、zoom、undo/redo E2E。

4. 账号和云端能力可能冲淡无账号定位。
   - 缓解：cloud capability 必须可选；每个云端 PR 都测试 signed-out local mode。

5. 依赖升级可能造成 UI 回归。
   - 缓解：小批次升级，配合截图、E2E 和 bundle 检查。
