# 安全政策

## 支持版本

当前重构分支以 `main` 为唯一安全维护基线。历史提交、旧 fork、未合并实验分支不承诺安全修复。

Phase 0 期间，本项目优先修复以下风险：

- 本地配置损坏导致页面白屏。
- SQL、DBML、JSON、DDB 导入缺少大小和复杂度限制。
- 分享功能的数据外发边界不清晰。
- Docker/nginx 缺少基础安全 header。
- 依赖审计中出现 high 或 critical 级别漏洞。

## 漏洞报告

如果发现安全问题，请不要在公开 issue 中披露可利用细节。建议通过项目维护者的私有渠道报告，并包含：

- 受影响的提交或版本。
- 复现步骤。
- 影响范围。
- 是否涉及用户图表数据、浏览器本地存储、分享后端或构建产物。
- 可行的缓解建议。

收到报告后，维护者应先确认影响范围，再决定是否公开 issue、发布补丁或记录风险接受说明。

## 隐私与数据边界

drawDB 的默认产品模式是本地优先：

- 无需账号即可使用核心编辑能力。
- 图表默认保存到当前浏览器的 IndexedDB。
- 设置项默认保存到当前浏览器的 `localStorage`。
- SQL、DBML、JSON、DDB 导入默认在浏览器内处理。
- 导出默认在浏览器内生成文件。

除非用户显式触发分享、云同步或其他网络功能，否则项目不应自动上传图表数据。

### Vercel Analytics

前端入口当前加载 `@vercel/analytics`，用于记录匿名化的页面访问和 Web Vitals 等产品运行指标。该遥测边界必须保持为产品可用性指标，不得上传图表内容、SQL/DBML/JSON/DDB 导入文本、IndexedDB 图表数据、`localStorage` 设置、分享 payload、云端 token 或用户输入的数据库连接信息。

如果后续新增自定义 analytics event，必须先确认事件字段不包含图表结构、表名、字段名、注释、默认值、错误堆栈中的图表内容或可反向识别个人/组织的数据。需要诊断导入、分享或云端错误时，应只上报粗粒度状态、错误类别和非内容型计数。

## 分享模式

分享是一个显式网络动作。用户点击分享并确认后，当前图表数据会发送到配置的后端服务，用于生成可访问链接。

实现分享功能时必须遵守：

- 未配置后端时，不应发起分享请求。
- 首次上传前必须向用户说明图表数据会发送到后端。
- 分享失败时不应破坏本地图表。
- 撤销分享应尽量删除或失效后端保存的数据。
- 日志和错误信息不得包含完整图表内容。

## 非可信输入

以下输入都必须按非可信数据处理：

- 用户上传的 SQL 文件。
- 用户粘贴的 SQL 文本。
- JSON 或 DDB 图表文件。
- DBML 文件。
- 分享链接返回的数据。
- 模板和后续云端同步数据。

导入前应限制文件大小、文本长度、对象数量、字段数量、关系数量和字符串长度。解析失败时应返回可理解的错误，不应清空当前图表。

## Dependency Audit 策略

常规门禁：

```bash
npm audit --audit-level=high
```

处理规则：

- high 或 critical 漏洞默认阻断合并。
- moderate 漏洞需要记录在 `docs/engineering/验证矩阵.md`，并说明是否有可行升级路径。
- 如果漏洞来自间接依赖且短期无法安全升级，需要记录风险接受原因和后续跟踪任务。
- 不为了消除 audit 告警而盲目进行大版本依赖升级。

## 安全开发要求

- 涉及输入解析、持久化、分享、导入导出、后端 URL 配置的变更必须配测试或 smoke 验证。
- 不在前端仓库提交密钥、token、私有后端地址或个人账号信息。
- 新增第三方脚本、CDN 资源或外部 iframe 前必须说明必要性和风险。
- Docker/nginx 配置变更需要验证构建可用性。
