# drawDB 导入导出支持范围

版本：2026-07-01

## 1. 目的

本文定义 Phase 2 导入导出可靠性工作的支持边界、fixture 命名规则和 warning/error issue 口径。后续 SQL、DBML、JSON、DDB、Markdown、Mermaid、image、PDF 的 import/export service 与 golden tests 均以本文为基线。

原则：

- 导入内容一律视为非可信输入，必须先经过大小、复杂度、结构和引用校验。
- 支持范围以 fixture-backed 能力为准；没有 fixture 的能力不得在产品文案中承诺为稳定支持。Phase 2.2 已建立 6 个 SQL dialect 的 `basic` fixtures；其余能力仍按下表规划推进。
- 部分支持必须返回 warning issue，不得静默丢弃关键 schema 信息。
- 导入失败不得清空或覆盖当前图；只有用户确认导入模式后才能写入当前 diagram。
- 导出结果必须稳定，适合 golden tests 做文本或结构回归。

## 2. 支持状态定义

| 状态 | 含义 | service 行为 |
| --- | --- | --- |
| 支持 | 已有 fixture 和测试覆盖，导入或导出结果可回归。 | `ok: true`，必要时附带 informational issue。 |
| 部分支持 | 能保留主要结构，但存在数据库特有属性降级或暂不表达。 | `ok: true`，必须返回 warning issue 说明丢失或降级内容。 |
| 暂不支持 | 识别到语法或结构，但当前不能可靠映射到 drawDB Domain Model。 | 可返回 `ok: false` error issue，或在可继续解析时返回 warning issue。 |
| 解析失败 | 输入不是合法格式，或 parser 无法恢复。 | `ok: false`，不得返回会覆盖当前图的 diagram。 |

Issue 建议字段沿用 Phase 1 结构化 issue：

- `id`：稳定、可测试的 issue id。
- `severity`：`info`、`warning`、`error`。
- `objectType`：`diagram`、`table`、`field`、`relationship`、`index`、`enum`、`type`、`source`。
- `objectId`：能定位到实体时填写；源文本级问题可为空。
- `messageKey`：面向 i18n 和测试断言的稳定 key。
- `message`：当前中文或英文展示文案。
- `fixHint`：可操作修复建议。

## 3. SQL Dialect 支持矩阵

### 3.1 MySQL

| 能力 | 导入目标 | 导出目标 | Phase 2 fixture |
| --- | --- | --- | --- |
| table | 支持 `CREATE TABLE` 基础字段列表。 | 输出稳定 `CREATE TABLE`。 | `mysql-basic.sql`、`mysql-basic.expected.json` |
| primary key | 支持字段级和表级 primary key。 | 输出字段级或表级 primary key，保持稳定顺序。 | `mysql-primary-key.sql` |
| composite foreign key | 支持表级复合外键，引用关系写入 `relationships`。 | 输出表级复合外键。 | `mysql-composite-foreign-key.sql` |
| index | 部分支持普通索引和唯一索引；全文、空间索引先 warning。 | 输出普通索引和唯一索引。 | `mysql-index.sql` |
| unique | 支持字段级和表级 unique。 | 输出 unique constraint 或 unique index。 | `mysql-unique.sql` |
| default | 部分支持字面量和常见函数；表达式默认值先 warning。 | 稳定输出已保留 default。 | `mysql-default.sql` |
| check | 部分支持；MySQL 版本差异不做语义判断。 | 保留为 check 文本。 | `mysql-check.sql` |
| comment | 支持表和字段 comment。 | 输出表和字段 comment。 | `mysql-comment.sql` |
| enum | 支持字段内 enum，后续可映射到 Domain enum。 | 输出字段 enum。 | `mysql-enum.sql` |
| unsupported syntax | partition、trigger、procedure、view 暂不支持。 | 不导出。 | `mysql-unsupported.sql` |

## 5. Fixture 目录与命名规则

目录：

- `src/test/fixtures/import-export/`：跨格式说明、共享 expected diagram、issue/golden 命名规则。
- `src/test/fixtures/sql/`：SQL dialect 输入、expected diagram 和 unsupported syntax cases。
- `src/test/fixtures/dbml/`：DBML 输入、expected diagram 和 golden output。
- `src/test/fixtures/diagrams/`：normalized diagram JSON、旧格式兼容样本、导出输入样本。

命名：

- SQL 输入：`<dialect>-<capability>.sql`
- SQL 期望结构：`<dialect>-<capability>.expected.json`
- SQL 导出 golden：`<dialect>-<capability>.golden.sql`
- DBML 输入：`dbml-<capability>.input.dbml`
- DBML 期望结构：`dbml-<capability>.expected.json`
- DBML 导出 golden：`dbml-<capability>.golden.dbml`
- JSON/DDB 输入：`<format>-<capability>.input.<ext>`
- JSON/DDB 期望结构：`<format>-<capability>.expected.json`
- 文档导出 golden：`<format>-<capability>.golden.<ext>`
- issue 期望：`<source>-<capability>.issues.json`

能力名使用 kebab-case，并优先从以下集合中选择：

- `basic`
- `primary-key`
- `composite-foreign-key`
- `index`
- `unique`
- `default`
- `check`
- `comment`
- `enum`
- `unsupported`
- `roundtrip`

## 6. Warning 与 Error 口径

返回 warning 的情况：

- 输入整体可导入，但某个数据库特有属性无法映射，例如 PostgreSQL expression index、MSSQL clustered index、Oracle tablespace。
- 字段 default/check/comment 可以作为文本保留，但 drawDB 暂不理解其语义。
- SQL 中出现 view、trigger、procedure 等不影响 table 导入的对象。
- schema-qualified name 被降级为 table name，schema 信息暂存到 comment 或 issue。

返回 error 的情况：

- 文件类型与内容不匹配，或 JSON/DDB/DBML/SQL 解析失败。
- 输入超出 Phase 0 import limits。
- 导入结果缺少必需 Domain Model 字段，且无法 normalize。
- 关系引用不存在，且无法安全降级为 warning。
- parser 抛出不可恢复错误，无法得到可展示 preview。

导入 service 输出要求：

```js
{
  ok: boolean,
  diagram: normalizedDiagramOrNull,
  preview: {
    tableCount: number,
    relationshipCount: number,
    typeCount: number,
    enumCount: number,
    warningCount: number,
    errorCount: number
  },
  issues: structuredIssues
}
```

`ok: false` 时 `diagram` 必须为 `null` 或不返回，调用方不得写入当前 React state。
